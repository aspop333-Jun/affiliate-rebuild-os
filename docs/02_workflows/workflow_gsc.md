# Workflow: Google Search Console (GSC)

## 1. 目的
*   記事がGoogleに認識（インデックス）されたかを確認する。
*   インデックスされない場合、その理由（除外理由）を特定し対策する。

## 2. 定期観測ルーチン
*   **頻度**: 週1回（例：毎週月曜）
*   **確認箇所**:
    *   「インデックス作成」→「ページ」レポート
    *   「未登録」の理由内訳を確認。

## 3. 除外理由別の対応方針

### A. 「クロール済み - インデックス未登録」
*   **意味**: Googleは来たが、「品質が低い」「価値がない」と判断して登録しなかった。
*   **対策**:
    *   記事のリライト（独自性アップ、AI臭削除）。
    *   内部リンクの拡充。
    *   **最優先対応**。

### B. 「検出 - インデックス未登録」
*   **意味**: 存在は知っているが、まだクロールすらしていない（クロール予算不足 or 優先度低）。
*   **対策**:
    *   「URL検査」から手動でリクエストを送る。
    *   トップページや人気記事からリンクを貼る。

### C. 「重複しています。送信されたURLが正規URLとして選択されていません」
*   **意味**: 他のページと内容が被っている。
*   **対策**:
    *   canonicalタグが正しく設定されているか確認。
    *   記事統合を検討。

## 4. 記録
*   重要な変化（急にインデックスが増えた/減った）があった場合は、`ops/decisions/` または各サイトの `gsc_notes.md` に記録を残す。
